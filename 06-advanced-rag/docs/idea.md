# Идея: RAG-ассистент на базе LangChain

## Описание проекта

RAG-ассистент (Retrieval-Augmented Generation) реализован в виде Telegram-бота, который использует базу знаний из документов для предоставления точных и релевантных ответов.

## Основная задача

Бот должен вести диалог с пользователем и отвечать на его вопросы, используя RAG-подход:
- Поиск релевантной информации в базе знаний (PDF-документы; при доработке по ДЗ модуля 4 — также JSON с готовыми парами вопрос–ответ)
- Генерация ответа на основе найденного контекста и истории диалога
- Понимание уточняющих вопросов с учётом контекста беседы (трансформация запроса перед retrieval)

## Ключевые особенности

- **Платформа**: Telegram
- **Технология**: RAG на базе LangChain (LCEL-цепочки)
- **База знаний**: PDF в `data/` (в учебном эталоне также используется JSON `sberbank_help_documents.json` в заданиях на расширение индекса)
- **Продвинутый retrieval** (модуль 6 — Advanced RAG):
  - Три режима поиска, переключаемые через конфиг: **semantic**, **hybrid** (Semantic + BM25), **hybrid+reranker** (+ Cross-Encoder)
  - Гибридный поиск объединяет векторный и keyword-поиск (RRF через `EnsembleRetriever`)
  - Cross-Encoder reranking для финального ранжирования чанков
  - Выбор провайдера эмбеддингов: OpenAI (OpenRouter) или HuggingFace (локально)
- **Функционал**:
  - Интеллектуальный поиск по документам (с настраиваемым режимом retrieval)
  - Контекстные ответы с учётом истории диалога
  - Трансформация уточняющих вопросов перед поиском
  - Управление индексацией документов
- **Поведение**: Ответы формируются на основе документов и контекста беседы
- **Мониторинг и качество**: трейсинг RAG-запросов в LangSmith, оценка ответов метриками RAGAS (команда `/evaluate_dataset`) и автоматический синтез тестовых Q&A-датасетов из документов (`make dataset` / `make dataset-upload`); сравнение режимов retrieval через evaluation
