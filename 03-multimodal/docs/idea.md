# Идея: персональный финансовый советник в Telegram

Telegram-бот — персональный финансовый советник, который ведёт учёт доходов и расходов на основе сообщений в чате.

## Что делает бот

- **Принимает записи о тратах и доходах** — в свободной форме текстом («потратил 450 на такси до аэропорта», «получил зарплату 120000»).
- **Принимает фото чеков** — извлекает из изображения сумму, дату, товары/услуги и прочие детали.
- **Ведёт таблицу транзакций** в памяти для каждого пользователя Telegram: дата, время, доход/расход, сумма (рубли), тип, категория, описание.
- **Отвечает на запросы о балансе** — по команде или естественной фразе («какой баланс?»); показывает общий баланс (доходы − расходы).

## Роль ассистента

Бот не просто сохраняет цифры, а ведёт себя как финансовый советник: подтверждает понимание записи, при необходимости уточняет неоднозначности и даёт краткие комментарии по запросу. Стиль задаётся системной инструкцией.

## Правила учёта

- **Валюта**: только рубли (RUB).
- **Пользователи**: у каждого Telegram-пользователя своя таблица транзакций.
- **Категории**: фиксированный список (продукты, рестораны, такси, образование, путешествия и др.).
- **Тип транзакции** (повседневные / периодические / разовые): определяет LLM по контексту сообщения.
- **Изменения**: только добавление записей; редактирование и удаление в MVP не предусмотрены.

## Приватность

Финансовые данные чувствительны. В продакшене должна быть возможность работать с **локальными моделями** (Ollama на GPU-сервере) вместо внешних API (OpenRouter), без изменения логики бота.

## Что сознательно не входит в MVP

- Банковские интеграции, экспорт в Excel, графики и разбивка по категориям в отчётах.
- Персистентное хранилище (БД, файлы) — на первом этапе данные в памяти процесса.
- Мультивалютность, редактирование и удаление транзакций.

Технические детали — в `docs/vision.md`.
