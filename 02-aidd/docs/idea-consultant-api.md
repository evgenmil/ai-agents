# Идея: LLM-консультант как HTTP-сервис

**Отдельный новый проект.** Сервис создаётся не внутри существующего приложения, а как самостоятельный микросервис.

Сервис призван отвечать на вопросы пользователей о событиях (мероприятиях, бронированиях). Внешнее приложение (например, PHP с webhook Telegram-бота) при получении сообщения от пользователя передаёт в сервис текст вопроса и контекст события. Сервис формирует ответ с помощью нейросети (LLM) и возвращает готовый текст. Внешнее приложение само отправляет этот ответ пользователю и решает, как и когда вызывать сервис.

**Сервис не дублирует бизнес-логику внешнего приложения:** пользователи, события, привязка к Telegram, отправка сообщений остаются во внешней системе. Микросервис выполняет только одно действие — по запросу (вопрос + контекст) сформировать ответ через LLM и вернуть его.

**Сервис не хранит историю диалога.** Каждый запрос обрабатывается отдельно. Если для связного диалога нужны предыдущие реплики, их передаёт внешнее приложение в теле запроса; само ведение истории — зона ответственности внешнего приложения.

Главная цель — добавить консультанта к уже существующему боту без переноса приёма апдейтов и без переноса доменной логики: один узкий слой генерации ответов по запросу.

Технические детали реализации на этапе идеи не фиксируются.
