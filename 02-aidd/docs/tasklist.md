## Итерация 1. Базовый Telegram‑бот

- [x] Бот отвечает на `/start` приветственным сообщением.
- [x] Бот отвечает простым эхо‑сообщением на любое текстовое сообщение.
- [x] Проверка: отправить несколько сообщений с разных устройств и убедиться, что ответы приходят стабильно.

## Итерация 2. Подключение LLM

- [x] Бот вместо эхо‑ответа формирует осмысленный текст с помощью LLM.
- [x] Сообщения пользователя передаются в модель, ответ возвращается в чат без дополнительной логики.
- [x] Проверка: задать боту простые вопросы и убедиться, что ответы связаны с вопросами.

## Итерация 3. Поддержка контекста диалога

- [x] Бот хранит для каждого пользователя несколько последних сообщений диалога.
- [x] Ответы учитывают предыдущие реплики в рамках одной сессии.
- [x] Проверка: вести непрерывный диалог из нескольких сообщений и убедиться, что бот помнит контекст.

## Итерация 4. Обработка ошибок и устойчивость

- [x] При ошибке LLM или Telegram бот не падает, а возвращает понятное сообщение пользователю.
- [x] Ошибки фиксируются в логах или консоли для дальнейшего разбора.
- [x] Проверка: искусственно спровоцировать ошибку (например, временно убрать ключ) и убедиться, что бот ведёт себя предсказуемо.

## Итерация 5. Конфигурация и переменные окружения

- [x] Критичные настройки (токены, ключи, модель) читаются из переменных окружения.
- [x] В коде не остаётся «зашитых» секретов и адресов внешних сервисов.
- [x] Проверка: изменить значения переменных и убедиться, что поведение бота корректно меняется.

## Итерация 6. Роли ассистента и стиль ответов

- [x] Определён базовый образ/роль ассистента (например, «дружелюбный технический помощник»).
- [x] Ответы бота соответствуют выбранному стилю и роли во всём диалоге.
- [x] Проверка: задать вопросы в разных контекстах и убедиться в единообразии стиля.

## Итерация 7. Логирование и простая диагностика

- [x] Включено базовое логирование ключевых событий (старт бота, входящие сообщения, ошибки).
- [x] Логи помогают понять, что происходит в боте без чтения кода.
- [x] Проверка: запустить бота, пообщаться с ним и убедиться, что в логах отражаются основные шаги.

## Итерация 8. Упаковка и запуск в Docker

- [x] Собран минимальный Docker‑образ для локального запуска бота.
- [x] Запуск бота в контейнере не отличается по функциональности от локального запуска.
- [x] Проверка: запустить бота в Docker и пройти базовый сценарий диалога из предыдущих итераций.

## Итерация 9. Индикатор набора сообщения

- [x] После получения сообщения пользователя бот показывает индикатор «печатает…» (chat action «typing») до отправки ответа.
- [x] Индикатор отображается в чате Telegram (иконка/подпись «печатает») на время ожидания ответа LLM.
- [x] Проверка: отправить сообщение боту и убедиться, что в чате видно индикатор набора перед появлением ответа.
