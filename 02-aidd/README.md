# Basic AI Assistant

Telegram-бот-ассистент на базе LLM. Ведёт диалог с пользователем, отвечает на вопросы и поддерживает контекст разговора. Стиль и тон ответов задаются системной инструкцией (system prompt). Минимальный MVP без оверинжиниринга — для быстрой проверки идеи и расширения при необходимости.

---

## Основные возможности

- **Диалог с LLM** — ответы формируются через модель (OpenRouter), а не эхо или шаблоны.
- **Поддержка контекста** — бот учитывает несколько последних сообщений в рамках сессии.
- **Роль и стиль ассистента** — настраиваемый образ (например, «дружелюбный технический помощник») через system prompt.
- **Команда `/start`** — приветственное сообщение при первом обращении.
- **Индикатор набора** — пока бот ждёт ответ от LLM, в чате отображается «печатает…».
- **Обработка ошибок** — при сбоях LLM или Telegram пользователь получает понятное сообщение, детали пишутся в логи.
- **Конфигурация через переменные окружения** — токены, ключи и модель задаются без изменения кода.

## Технологический стек

| Компонент | Технология |
|-----------|------------|
| Язык | Python 3.11+ |
| Зависимости | `uv` (pyproject.toml, uv.lock) |
| LLM | Официальный клиент `openai`, провайдер **OpenRouter** |
| Telegram | **aiogram** (режим polling) |
| Запуск | `make run` / Docker (python:3.11-slim) |
| Состояние | В памяти процесса (без БД на этапе MVP) |

Структура: `src/`-layout, пакеты `basic_ai_assistant`, `telegram_bot`, `llm`, `dialog`. Подробнее — в [docs/vision.md](docs/vision.md).

## Установка и запуск

### Требования

- Python 3.11+
- [uv](https://docs.astral.sh/uv/) (менеджер зависимостей и окружений)

### 1. Клонирование и зависимости

```bash
git clone <url-репозитория>
cd basic-ai-assistant
make install
```

Или вручную: `uv sync` (из корня проекта).

### 2. Переменные окружения

Создайте файл `.env` в корне проекта — при `make run` он подхватывается автоматически (переменные, уже заданные в shell, имеют приоритет). Альтернатива: экспорт в shell:

```env
TELEGRAM_BOT_TOKEN=ваш_токен_бота
OPENROUTER_API_KEY=ваш_ключ_openrouter
OPENROUTER_BASE_URL=https://openrouter.ai/api/v1
LLM_MODEL=openai/gpt-3.5-turbo
LOG_LEVEL=INFO
SYSTEM_PROMPT_PATH=prompts/system_prompt.txt
```

- `TELEGRAM_BOT_TOKEN` и `OPENROUTER_API_KEY` — обязательны.
- `OPENROUTER_BASE_URL` и `LLM_MODEL` — можно не задавать, тогда подставятся значения по умолчанию.
- **Системный промпт** (роль и стиль ассистента):
  - рекомендуется `SYSTEM_PROMPT_PATH` — путь к файлу (относительно корня проекта или абсолютный), например `prompts/system_prompt.txt`;
  - альтернатива: `SYSTEM_PROMPT` — одна строка в `.env` (удобно только для коротких текстов; для Docker `--env-file` многострочные значения не поддерживаются);
  - если обе переменные не заданы — используется встроенный промпт по умолчанию.
- Текст промпта можно править в `prompts/system_prompt.txt` без пересборки образа (при `make docker-run` каталог монтируется в контейнер).

### 3. Запуск локально

```bash
make run
```

Эквивалент: `PYTHONPATH=src uv run -m basic_ai_assistant.main`

### 4. Запуск в Docker

```bash
make docker-build
make docker-run
```

Для `docker-run` переменные окружения подхватываются из `.env` (флаг `--env-file .env`), каталог `prompts/` монтируется в контейнер. В `.env` укажите `SYSTEM_PROMPT_PATH=prompts/system_prompt.txt` (в контейнере путь тот же: `/app/prompts/system_prompt.txt`). Убедитесь, что `.env` не попал в репозиторий (должен быть в `.gitignore`).
