# Отчёт: RAG-ассистент Сбербанка

Telegram-бот на LangChain: отвечает по PDF (кредиты, вклады) и JSON-FAQ (карты), ищет релевантные фрагменты, подставляет их в промпт LLM, помнит историю диалога.

**Вариант:** Лайт (эталон `live/` + доработки по ДЗ-4).

## Что сделано

Telegram (aiogram), OpenRouter, индексация PDF + JSON в `InMemoryVectorStore`, RAG с query transformation, команды `/start`, `/help`, `/index`, `/index_status`, логи и отладка retrieval (`RAG_DEBUG`).

## Стек и модели

Python 3.11, uv, LangChain, PyPDF, JSONLoader + jq.

| Роль | Модель |
|------|--------|
| Ответы и transform | `google/gemini-3-flash-preview` |
| Эмбеддинги | `openai/text-embedding-3-small` |

`RETRIEVER_K=3`. Подробные логи экспериментов: `docs/chunk-size-testing.md`, `docs/rag-retrieval-debugging.md`.

## Эксперименты

### Чанкинг PDF (`RecursiveCharacterTextSplitter`)

Проверяли 300/30, **800/80**, 1500/150 на трёх вопросах (аннуитет, изменение договора, вклады).

- **300** — ответы размытые, по вкладам расходятся с документом.
- **800** — лучший баланс на всех вопросах → **оставили в проекте**.
- **1500** — чуть богаче на аннуитете, но слабее на договоре и вкладах.

### JSON

`JSONLoader` с `jq_schema=".[].full_text"`: один Q&A = один вектор, **без чанкинга**. PDF режутся на чанки, всё склеивается в `indexer_with_json.py`. Дубликаты в JSON забивают top-k — это видно в логах.

### Эмбеддинги (OpenRouter)

| Модель | Итог |
|--------|------|
| `text-embedding-3-large` | Работает стабильно |
| `text-embedding-3-small` | **Выбрали:** дешевле, качество как у large |
| `qwen/qwen3-embedding-8b` | Не подошла — пустые ответы |

## Главные инсайты

1. **RAG чаще ломается на поиске, а не на LLM.** Вопрос был в базе, по исходной формулировке чанк был на 1-м месте — но query transform переписал запрос в «обзор про доставку по России», в top-3 попали соседние FAQ, и модель честно ответила «не нашёл». Без логов retrieval это выглядит как «плохая модель».

2. **Размер чанка реально влияет на смысл ответа** — не только на число чанков в индексе. 800 оказался sweet spot для смешанных PDF.

3. **Дорогая embedding-модель не обязательна** — small сравнялась с large; зато «красивая» qwen на практике не заработала — модель эмбеддингов надо проверять на своих данных.

4. **FAQ лучше индексировать целиком**, а transform для коротких вопросов держать близко к тексту пользователя.
